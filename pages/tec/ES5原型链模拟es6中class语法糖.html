以下是javascript总结笔记之原型链一

  1. 通过电子产品类(eleProduct)创建电脑类(pc)，通过电脑类(pc)创建苹果电脑实例(mac)
  2. 使用es6的class语法糖实现这个继承关系
  3. 使用es5的原型链实现这个继承关系
  4. 总结原型链的特性

第一步：使用ES6的class创建对象并展现继承关系

class EleProduct {
    constructor(hasScreen) {
        this.hasScreen = hasScreen;
    }

    getWeight() {
        return 'weight is 2kg';
    }
}

class Pc extends EleProduct {
    constructor(hasScreen, screenSize) {
        super(hasScreen);
        this.screenSize = screenSize;
    }

    getDetail() {
        return 'pc can work';
    }
}


const mac = new Pc();
console.log('hasScreen' in mac);// true
console.log('screenSize' in mac);// true
console.log(mac instanceof Pc);// true
console.log(mac instanceof EleProduct);// true

ES6中的class有没有改变javascript中对象的工作机制（原型链）呢？

首先看MDN上面的解释：class 声明创建一个基于原型继承的具有给定名称的新类

第二步：用ES5的语法实现上述两个’类‘对象一个实例对象的创建

首先明确对象和函数的一个特性

  1. 每个对象都有一个内置的原型属性[[prototype]]，该属性指向这个对象的原型,此属性无法直接访问
  2. 每个函数都有一个原型对象，原型对象的constructor属性指向该函数本身，如：
const fun = function () { };
console.log(fun.prototype.constructor === fun);//true
  • 使用ES5语法实现原型链的继承
function EleProduct(hasScreen) {
  this.hasScreen = hasScreen;
}
EleProduct.prototype.getWeight = function getWeight() {
  return 'weight is 2kg';
}1⃣️

function Pc(screenSize) {
  this.screenSize = screenSize;
}

Pc.prototype = new EleProduct();2⃣️
mac = new Pc();3⃣️
console.log('hasScreen' in mac);//true
console.log('getWeight' in mac);//true
console.log('screenSize' in mac);//true
  • 在（1）处我们对EleProduct函数的原型属性添加了一个getWeight方法
  • 在（2）处我们将EleProduct的实例赋值给Pc的原型，这也是一种比较稳妥的实现原型链的方法
    1. 实现了Pc继承EleProduct
    2. 此时的一个副作用就是Pc之前包含constructor属性的原型对象断开了和Pc之间的关系
  • 在（3）处我们通过new关键字实例化了一个对象mac，此时发生的事情包括
    1. mac对象的原型属性[[prototype]]指向了Pc的原型对象
    2. 此时Pc作为构造函数，其上下文被定义为mac
    3. 此时Pc构造函数内部的this指向了mac

这样我们就把三个属性'hasScreen'，'getWeight'，'screenSize'完全继承下来了

但是如何解决上面的副作用呢？

我们首先看副作用的表现

// 接着ES5实现的原型链继续探索
console.log(mac instanceof Pc);//true
console.log(mac.constructor === Pc);4⃣️//false
console.log(mac.constructor === EleProduct);5⃣️//true
  • 在（4）处和我们之前的结论：mac继承自Pc,mac本身没有constructor属性，其沿着原型链向上寻找
  • 到了Pc处，Pc的原型和之前带有constructor断开了联系，因此此时（4）处是false
  • 继续向上寻找是否存在constructor，此时EleProduct的原型存在constructor，其值就是EleProduct本身

总结：

  • 我们虽然完成了ES5代码实现一个class语法糖，但是原型链的知识远远不止如此
